# Azure Static Web Apps Deployment Guide for Dheeraj Krishna Enterprises

## 1. Prerequisites
- GitHub account
- Azure account (Free Tier is sufficient)
- [Git installed](https://git-scm.com/downloads)
- [Node.js installed](https://nodejs.org/) (for local testing, optional)

## 2. Project Structure
```
my-seeds-shop-website/
  |-- index.html
  |-- styles.css
  |-- script.js
  |-- assets/
```

## 3. Push Code to GitHub
1. Create a new repository on GitHub (e.g., `my-seeds-shop-website`).
2. Initialize git in your project folder:
   ```sh
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin https://github.com/<your-username>/my-seeds-shop-website.git
   git push -u origin main
   ```

## 4. Deploy to Azure Static Web Apps
1. Go to [Azure Portal](https://portal.azure.com/)
2. Search for **Static Web Apps** and click **Create**
3. Fill in:
   - **Subscription**: Free Tier
   - **Resource Group**: Create new or use existing
   - **Name**: my-seeds-shop-website
   - **Region**: Central India (or closest)
   - **Hosting plan**: Free
   - **Source**: GitHub
   - **Repository**: Select your repo
   - **Branch**: main
   - **Build Presets**: Custom
   - **App location**: `/` (root)
   - **Api location**: (leave blank)
   - **Output location**: `/` (root)
4. Click **Review + Create** and then **Create**
5. Authorize GitHub if prompted
6. Azure will set up a GitHub Actions workflow automatically

## 5. Verify Deployment
- After a few minutes, your site will be live at `https://<your-app-name>.azurestaticapps.net`
- Test on mobile and desktop

## 6. Updating the Site
- Make changes locally
- Commit and push to GitHub
- Azure will auto-deploy your changes

## 7. Accessibility & Quality Checklist
- ✅ Telugu font renders correctly
- ✅ Language toggle works instantly
- ✅ Loads under 2 seconds
- ✅ Buttons are large and accessible
- ✅ Contrast is readable
- ✅ Google Maps embed works
